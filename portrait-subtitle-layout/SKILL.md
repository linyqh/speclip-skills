# 竖屏视频字幕位置与参数

用于 **竖屏视频** 的字幕排版与烧录决策。核心目标是解决竖屏里最常见的问题：字幕贴底、挡住口播人物下巴/胸口、压住底部 CTA、两行太长、信息密度过高。

## 什么时候用

- 用户明确说是 **竖屏 / 9:16 / portrait** 视频
- 用户要字幕烧录，并且关心 **位置、安全区、字号、边距**
- 用户说：**别挡人脸 / 再往上抬一点 / 底部还有按钮 / 字幕两行太满**
- 任务涉及短视频平台内容，如抖音、快手、小红书、Reels、Shorts

如果是横屏 / 16:9，请改用 `landscape-subtitle-layout`。

## 默认基准

以 **1080x1920** 竖屏为基准，默认采用 **底部居中但明显上抬** 的字幕：

- `Alignment = 2`（bottom-center）
- `FontSize = 80`
- `MarginV = 600`
- `MarginL = 54`
- `MarginR = 54`
- `Outline = 3`
- `Shadow = 1`
- `WrapStyle = 2`
- `MaxLineLength = 14`
- 目标：**最多两行**，单行更短，避免形成大块字幕墙

这套参数与 Speclip 当前内置竖屏字幕基线保持一致，默认就比横屏高很多，用来避开底部操作区、贴纸、标题条和平台 UI。

## 缩放公式

如果分辨率不是 1080x1920，按下面规则缩放：

```text
fontSize = clamp(round(80 * H / 1920), 52, 140)
marginV  = clamp(round(600 * H / 1920), 320, 1200)
marginL  = clamp(round(54 * W / 1080), 36, 120)
marginR  = clamp(round(54 * W / 1080), 36, 120)
outline  = clamp(round(3 * H / 1920), 2, 5)
shadow   = clamp(round(1 * H / 1920), 1, 2)
```

其中：

- `W` = 视频宽度
- `H` = 视频高度

## 位置调整规则

### 1) 默认竖屏位置

- 保持 `Alignment = 2`
- 常规内容优先 `MarginV = 560–680`
- 默认推荐值是 `600`

### 2) 用户说“别挡到底部按钮 / 商品卡 / CTA”

优先把 `MarginV` 调到：

- `680–820`：底部有明显文案区或贴纸时
- `820–960`：底部长期存在按钮、进度条、商品卡时

### 3) 用户说“字幕太高了，想压低一点”

优先把 `MarginV` 调到：

- `480–560`：轻度下移
- `420–480`：明显下移，但要确认底部没有 UI 或装饰元素
- 一般不建议低于 `420`

### 4) 用户说“别挡脸 / 别挡嘴部 / 别挡人物胸口”

优先处理顺序：

1. 先上移 `MarginV`
2. 再缩短每行长度
3. 再考虑略微缩字号

如果是口播/对镜视频，推荐：

- `MarginV = 680–860`
- `MaxLineLength = 12–14`

### 5) 行太长、两行太满

- 默认 `MaxLineLength = 14`
- 节奏快、短句强口播：`12–14`
- 轻叙述类：`14–16`
- 尽量拆句，不要先缩到很小字号

## ASS 建议参数

标准样式行可直接按下面结构生成：

```text
Style: Default,PingFang SC,80,&H00FFFFFF,&H000000FF,&H00000000,&H66000000,-1,0,0,0,100,100,0,0,1,3,1,2,54,54,600,1
```

字段重点：

- `Outline = 3`
- `Shadow = 1`
- `Alignment = 2`
- `MarginL = 54`
- `MarginR = 54`
- `MarginV = 600`

竖屏内容更容易出现复杂背景和人像主体，因此描边通常比横屏更厚一点。

## 推荐工作流

1. 先拿到 `.srt`
2. 转成 `.ass`
3. 按竖屏参数修改样式
4. 再烧录到最终视频

如果用户只说“帮我做短视频字幕”，优先按竖屏安全区思路处理，而不是简单贴在最底部。

## FFmpeg 烧录示例

```bash
ffmpeg -y -i "IN.mp4" -vf "subtitles='SUB.ass'" \
  -c:v libx264 -preset medium -crf 19 \
  -c:a aac -b:a 128k -map 0:v:0 -map 0:a? -movflags +faststart "OUT.mp4"
```

## 常见决策

### 口播 / talking head

- 优先抬高字幕，避开嘴部与下巴区域
- `MarginV = 680–860`
- `MaxLineLength = 12–14`

### 带底部 CTA / 商品卡 / 标题条

- `MarginV = 760–960`
- 必要时进一步缩短单行长度

### 节奏很快的剧情解说

- 保持两行内
- 用更短句子，不要让单屏字幕停留过长
- 默认 `FontSize = 80` 已足够，大多数时候不需要放大

## 输出要求

使用这个 skill 时，最终回答应包含：

1. 当前判定为竖屏的依据（分辨率或用户说明）
2. 最终采用的字幕参数
3. 是否为避让人物 / CTA / 底部 UI 而上移
4. 可直接使用的 `.ass` 样式或 FFmpeg 命令
5. 输出路径或建议输出文件名
