# drama-explainer Review Rubric

用于对 `drama-explainer` 的输出进行人工 review 和 A/B 对比。

## 总分建议

- 90-100：高质量，可直接交付
- 70-89：合格，有少量细节可优化
- 50-69：勉强可用，但明显缺戏
- 0-49：结构能跑通，但成片质量差

## 评分维度

### 1. 内容（1-5）
- 剧情准确：人物关系、关键事件、时序是否准确
- 戏眼保留：最关键的羞辱、反抗、安慰、反转、站队是否保留
- 情绪链条：是否保留完整情绪推进，而非只保留结果

### 2. 编辑（1-5）
- 原片选段力度：选中的是不是“最有戏”的原片
- VO 克制程度：VO 是否只做铺垫、转场、补信息
- 候选召回质量：若环境具备向量检索，是否先召回关键戏眼再做筛选
- 节奏推进：是否存在拖沓、重复、信息已说过又再说一遍

### 3. 结构（1-5）
- `storyboard.json` 完整性：字段齐全、可直接执行
- 派生一致性：`final_script.md`、`voiceover.md` 是否与 `storyboard.json` 对齐
- 时间顺序合理性：同一 source 顺序是否合理，回跳是否有说明

### 4. 观感（1-5）
- 记忆点：是否留下能让人记住的句子或场面
- 代入感：是否真的让观众进入人物处境
- 模板化程度：是否避免短视频万能包装句

## 一票否决项

出现以下任一项，整体评分不得高于 49：
- `original.text` 是摘要标签，不是真实展示台词
- 漏掉最关键的高张力原片段
- VO 大量复述紧邻原片已说清的内容
- 结果能看懂，但没有任何明显戏眼

## Review 模板

```markdown
# Review

## Overall Score
- total: 45/100

## Rubric
- 内容：2/5
- 编辑：2/5
- 结构：3/5
- 观感：2/5

## Strengths
- 跑通了完整流程
- 基本剧情没有跑偏

## Weaknesses
- 原片戏眼选得不够狠
- VO 过于模板化
- 只保留结果，没有保留关键情绪铺垫
- `original.text` 有摘要化倾向

## Must Fix
- 补保留孩子安慰、婚宴羞辱等关键戏眼
- 压掉空泛 VO
- 强制 `original.text` 使用真实展示台词
```
