# 步骤 6：Dramatic Editor 指南

这份指南专门服务于步骤 6 的 subagent。它不是格式说明，而是**编辑决策说明**。

## 子任务目标

给定：
- `characters.md`
- `plot_analysis.md`
- `draft_script.md`
- `transcript*.json` / `*.srt`
- 必要时补充 `visual_analysis*.json` 的高张力场景摘要

输出：
- 一份高质量 `storyboard.json`
- 并可派生 `final_script.md` 与 `voiceover.md`

## 编辑优先级

从高到低：
1. 戏眼是否命中
2. 情绪链条是否完整
3. VO 是否克制
4. 时间顺序是否易于观看
5. 信息是否完整

## 先挑什么原片

优先挑这些：
- 最羞辱人的一句
- 最能立住主角脾气的一句
- 最能体现人物尊严被踩或反击成功的一句
- 最能表现母子情 / 夫妻站队 / 情绪反转的一句
- 演员表情、停顿、动作明显比字面信息更强的一句

不要优先挑这些：
- 只是把剧情说明白的一句
- 语义完整，但表演没戏的一句
- 可以被一条 VO 更高效带过的一句

## VO 怎么写才算合格

合格的 VO：
- 补前因后果
- 帮观众跳场景
- 在原片没讲清楚时补一点必要信息

不合格的 VO：
- 把原片后面要说的话提前说掉
- 复述原片刚讲过的内容
- 大量使用“命运、站稳、彻底改写人生、移不开眼”这种万能短视频句式
- 写得像标题党，不像解说剪辑稿

## 情绪链条检查

每次完成 storyboard 后，必须人工自查：
- 有没有“羞辱”
- 有没有“反抗”
- 有没有“落难”
- 有没有“安慰”
- 有没有“反转”
- 有没有“站队 / 护短”

不是每个故事都必须全有，但核心链条不能只剩结果。

## 审稿问题清单

输出前逐项问自己：
1. 我保留的是最有戏的一句，还是最方便的一句？
2. 这一段如果去掉原声，会不会明显变弱？
3. 这句 VO 是必要信息，还是原片已经自己讲清楚了？
4. 这条线有没有从“发生了什么”升级到“观众会不会被打到”？
5. 如果观众只看这 60-90 秒，会不会记住至少一个场面？

## 合格输出示意

- `original.text` 直接写真实台词
- `notes` 写保留原声的理由
- 有钩子前置时明确标记
- VO 数量少于你第一反应想写的数量
- 关键反转前有足够铺垫，不是突然告诉观众结果
