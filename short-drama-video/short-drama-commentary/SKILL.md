# 短剧解说结构设计

这个 skill 解决的不是“把剧情复述清楚”，而是：

**把剧情信息、原片戏眼、解说节奏，组织成一条能剪、能播、能打人的短剧解说结构。**

如果最后只有剧情梗概，没有明确的“哪里保留原片、哪里用解说带过、为什么这么插”，那就不算完成。

## 适用输入

优先处理下面三类输入：

1. **优秀短剧解说字幕样本**：用于学习结构、总结模式、复刻方法
2. **原剧字幕 / transcript**：用于直接设计一条新的短剧解说结构
3. **已有解说稿 / 旧版本成片字幕**：用于诊断节奏问题，重做原片与解说的穿插方案

如果只有视频没有字幕，本 skill 不是首选；先去补字幕或转录。

## 先判断当前任务属于哪一类

### A. 样本拆解
当用户给你的是“优秀短剧解说字幕”时：
- 先识别叙述声部和原片对白声部
- 再拆它的结构骨架、转场方式、原片插入点
- 最后总结成可复用规则，而不是只做内容概括

### B. 新稿规划
当用户给你的是“原剧字幕 / 梗概”时：
- 先梳理人物、冲突、反转、情绪链条
- 再决定哪些信息必须由解说压缩
- 哪些节点必须交给原片原声承担
- 最后输出一份可直接写稿或剪辑的插片方案

### C. 改稿提纯
当用户给你的是“已有解说稿 / 成片字幕”时：
- 先找节奏重复、原片太少、原片太碎、解说抢戏等问题
- 再重排为更强的“解说推进 + 原片打点”结构

## 核心判断标准

### 必须保留原片的地方
优先把这些位置交给原片，而不是解说代讲：

1. **一句话就能立住人物**
   - 比如狠话、回怼、拒绝、摊牌、护短、宣战
2. **羞辱感最强的瞬间**
   - 当众贬低、冤枉、揭短、逼迫、赶走、羞辱身份
3. **反击快感最强的瞬间**
   - 掀桌、打脸、反将一军、公开站队、身份揭晓
4. **情绪落点最强的瞬间**
   - 哭着放手、孩子安慰、求婚、认亲、诀别
5. **表演强于信息的瞬间**
   - 停顿、眼神、咬牙、颤抖、沉默后开口

### 应该交给解说的地方
以下内容优先用解说压缩，而不是硬塞原片：

1. **跨场景、跨时间的推进**
2. **重复性的羞辱或重复争吵**
3. **观众需要知道，但原片说得不够利索的背景**
4. **从一个爆点跳到下一个爆点之间的桥段**
5. **为了保节奏必须快速带过的说明段**

## 默认结构骨架

大多数短剧解说都能落进下面这条 6 段式骨架：

1. **开场钩子**
   - 先抛最危险、最屈辱、最反常、最有代价的一幕
   - 可以是 1 句解说，也可以直接上 1 句爆炸原台词
2. **快速补前情**
   - 用解说把人物关系、当前困境、事件前因讲清
   - 目标是尽快把观众送到第一个冲突点
3. **第一次正面碰撞**
   - 第一次插入关键原片
   - 常见是羞辱、误会、命令、拒绝、遇见、威胁
4. **连续升级**
   - 解说负责压缩过程
   - 原片只在升级节点打点，不要每个对话都留
5. **反转 / 反击 / 情绪爆点**
   - 原片占比提高
   - 这里往往是整条视频最值得保留原声的区域
6. **结尾落点 / 下集钩子**
   - 要么把最狠一句留给原片
   - 要么用 1 句解说把悬念挂出去

## 默认节奏建议

如果没有更明确要求，优先使用这个默认比例：

- **解说承担 60%~75% 的信息推进**
- **原片承担 25%~40% 的情绪打点**

再按题材微调：

- **家庭狗血 / 婚恋复仇**：解说 65% 左右，原片 35% 左右
- **战神 / 逆袭 / 爽文权势流**：原片可提高到 40%~55%
- **纯甜宠 / 治愈慢热**：解说可提高到 75%~85%

## 样本拆解流程

1. 先通读字幕，不急着总结剧情
2. 标出所有“原片值得保留”的点：
   - 羞辱
   - 回怼
   - 站队
   - 认亲
   - 求婚
   - 身份揭晓
   - 最后一刀
3. 再看这些点前后，解说都做了什么：
   - 是铺垫
   - 是补背景
   - 是跳时间
   - 是解释后果
4. 总结样本属于哪一型：
   - 纯解说
   - 解说主导 + 点状原片
   - 解说主导 + 关键对白
   - 解说对白混编
   - 对白主导 + 少量串讲
5. 最后沉淀成“可复用规则”，不要停在“这个剧情很抓马”

## 新稿规划流程

1. 先列出剧情主线，不超过 6 个大拍点
2. 每个拍点判断：
   - 观众此刻最需要知道什么
   - 这一拍最值得听见什么
3. 只把“听见会更爽/更疼/更有记忆点”的内容留给原片
4. 其余内容交给解说带过
5. 确保每次插原片都在承担任务，而不是装饰

## 不要这样做

- 不要把整段原片对话按顺序全抄进来
- 不要解说刚说完，原片又把同样的话说一遍
- 不要只保留结果，不保留羞辱和反击过程
- 不要只因为一句台词“语义完整”就保留它
- 不要把解说写成空泛短视频腔，比如“命运齿轮开始转动”
- 不要硬把每 10 秒都塞一个原片

## 输出合同

无论是拆样本还是做新稿，默认都要给出这三层结果：

### 1. 结构判断
至少说明：
- 当前素材属于哪种结构型
- 核心情绪链条是什么
- 原片承担什么，解说承担什么

### 2. 插片计划表
至少包含这些列：
- `段落`
- `模式`（`解说` / `原片`）
- `作用`
- `内容摘要`
- `为什么此处要保留原片 / 为什么此处要用解说`
- `时长建议`

### 3. 可执行文案或分段建议
按用户需求返回：
- 解说逐行稿
- 解说 + 原片混排脚本
- beat sheet
- 简版 storyboard 表

如果需要格式模板，读取 `references/output-template.md`。

## 质量自检

交付前逐项检查：

1. 开头是不是足够快进入冲突
2. 原片是不是都插在“最有戏”的地方，而不是“最好解释”的地方
3. 解说有没有重复紧邻原片已经说清的内容
4. 中段有没有拖沓的说明堆积
5. 结尾是不是留下了真正的情绪落点或悬念
6. 如果观众只看这 60~90 秒，会记住哪一句原台词、哪一个场面
