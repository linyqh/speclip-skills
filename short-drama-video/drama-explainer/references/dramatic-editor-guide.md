# 步骤 6：Dramatic Editor 指南

这份指南专门服务于步骤 6 的 subagent。它不是格式说明，而是**编辑决策说明**。

开始前先读：

- `../../references/excellent-short-drama-commentary-playbook.md`

## 子任务目标

给定：
- `characters.md`
- `plot_analysis.md`
- `draft_script.md`
- `transcript*.json` / `*.srt`
- 必要时补充 `visual_analysis*.json` 的候选片段复核摘要
- 如已生成，附上 `retrieval_candidates.json`

输出：
- 一份高质量 `storyboard.json`
- 并可派生 `final_script.md` 与 `voiceover.md`

## 检索先行，不要盲扫

dramatic editor 不是重新从零翻全量 transcript 的“第二个摘要器”。

默认流程应是：
1. 先消费 `videosearch` 召回的候选片段，或主代理提供的 `retrieval_candidates.json`
2. 再回到 transcript / SRT 校验真实台词和精确时间
3. 只有在难以判断表演强度、候选互相冲突、或存在无台词强情绪镜头时，再结合 `visual_analysis*.json` 做定向视觉复核

检索负责**召回**，不是直接替你做最终决策。

视觉复核负责**补足 transcript 看不见的表演信息**，不是替代检索重新从零找戏。

若候选不够好，应先补检索，而不是直接退回全量盲扫。

## 编辑优先级

从高到低：
1. 戏眼是否命中
2. 情绪链条是否完整
3. VO 是否克制
4. 检索候选是否覆盖关键戏眼
5. 节奏是否够快、够狠
6. 时间顺序是否易于观看
7. 信息是否完整

## 节奏预算

进入正式编辑前先确认：

- 开头第一刀是否能在 0-8 秒内落下
- 目标原片比例是否合理
- 有几段原片会超过 15 秒
- 结尾到底是哪一种策略

如果这些都没想清楚，先别开始切。

## 先挑什么原片

优先挑这些：
- 最羞辱人的一句
- 最能立住主角脾气的一句
- 最能体现人物尊严被踩或反击成功的一句
- 最能表现母子情 / 夫妻站队 / 情绪反转的一句
- 演员表情、停顿、动作明显比字面信息更强的一句

不要优先挑这些：
- 只是把剧情说明白的一句
- 语义完整，但表演没戏的一句
- 可以被一条 VO 更高效带过的一句

并且记住：

- 原片不是越长越高级
- 一整段都好看，不等于每一句都必须留下
- 只留“最该听见”的那几句，通常比完整保留更像成熟短剧解说

实际操作时，先检查检索候选里是否已经召回这些类型；如果没有，先补 query，再继续编辑。只有在“候选都差不多”或“字面一般但怀疑表演很强”时，再补视觉复核。

## VO 怎么写才算合格

合格的 VO：
- 补前因后果
- 帮观众跳场景
- 在原片没讲清楚时补一点必要信息

不合格的 VO：
- 把原片后面要说的话提前说掉
- 复述原片刚讲过的内容
- 大量使用“命运、站稳、彻底改写人生、移不开眼”这种万能短视频句式
- 写得像标题党，不像解说剪辑稿

## 情绪链条检查

每次完成 storyboard 后，必须人工自查：
- 有没有“羞辱”
- 有没有“反抗”
- 有没有“落难”
- 有没有“安慰”
- 有没有“反转”
- 有没有“站队 / 护短”

不是每个故事都必须全有，但核心链条不能只剩结果。

同时补做一次检索覆盖自查：
- 这些关键情绪节点是否都至少被召回过一次候选
- 最终选中的 `original` 是否能回溯到明确候选或明确人工校验依据

## 审稿问题清单

输出前逐项问自己：
1. 我保留的是最有戏的一句，还是最方便的一句？
2. 这一段如果去掉原声，会不会明显变弱？
3. 这句 VO 是必要信息，还是原片已经自己讲清楚了？
4. 我有没有因为检索 top1 分数高，就偷懒选了“相关但没戏”的片段？
5. 这条线有没有从“发生了什么”升级到“观众会不会被打到”？
6. 如果观众只看这 60-90 秒，会不会记住至少一个场面？
7. 这条更像“短剧解说”，还是“原剧高光串起来”？

## 合格输出示意

- `original.text` 直接写真实台词
- `notes` 写保留原声的理由
- 有钩子前置时明确标记
- VO 数量少于你第一反应想写的数量
- 关键反转前有足够铺垫，不是突然告诉观众结果
